UCIe 协议栈架构

理解 UCIe 的分层架构是制定验证计划的基础，每层都有独立的验证重点

📦 Protocol Layer（协议层）
支持多种协议映射，包括 PCIe、CXL、AXI 和 CXS，负责协议特定事务的处理、封装/解封装及排序规则。
PCIe TLP CXL.io CXL.mem CXL.cache AXI4/AXI5 ACE Coherency CXS Multi-Protocol
⬇   D2D Adapter   ⬇
🔄 D2D Adapter Layer（Die-to-Die 适配层）
核心链路管理层，负责链路初始化、参数协商、基于 Credit 的流控、CRC/重传及错误恢复机制。是 UCIe 最复杂的部分。
Link Training Param Exchange Flow Control CRC & Retry Error Recovery Sideband MSG Lane Repair FLIT Format
⬇   Physical Interface   ⬇
⚡ Physical Layer（物理层 + PHY）
基于 PCIe 电气规范，支持多数据速率（8/16/32 GT/s），负责链路训练、均衡、Lane 操作及电气特性验证。
8/16/32 GT/s Link Training EQ Phase 0-3 Lane Reversal Polarity Invert Width Negotiation Deskew BER Testing
