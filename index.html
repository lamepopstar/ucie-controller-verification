01. UCIe 协议栈架构分析

从底层物理链路到上层协议映射，逐层拆解验证对象。

📦

Protocol Layer

  • PCIe 协议映射（Flit/Non-Flit）
  • CXL.io / CXL.mem / CXL.cache 支持
  • AXI-4 / CXS 协议桥接
  • 事务排序、Req/Resp/Cmp 路由
  • VC 仲裁与反压管理
🔄

D2D Adapter Layer

  • 链路训练（Link Training）与状态机
  • 数据包封装、Parity / CRC 生成
  • 重传机制（Retry / ACK/NAK）
  • 流控与远程 Stalldet
  • 边带（Sideband）消息处理
⚡

Physical Layer

  • Mainband / Sideband 电气接口
  • 8 / 16 / 32 GT/s 速率训练
  • Lane 反转、极性反转、Lane 修复
  • 均衡（EQ）、眼图训练
  • 时钟、Reset、PM 状态同步
🛡️

System 支持

  • 电源管理（L0 / L1 / L2 / Retrain）
  • 错误注入、检测与恢复
  • 寄存器 CSR 配置与中断
  • 性能压力与带宽延迟测试
  • 复位、热插拔、多 Die 拓扑

05. Corner Case 矩阵

按场景分类的典型边界与异常用例，确保验证不留死角。

🔗 链路训练

  • 训练中途 Reset/掉电
  • 两端参数不一致协商
  • 连续训练失败达到上限
  • Sideband 训练失败 fallback
  • 极性/顺序反转组合

⚡ 物理层

  • 最高速率下高抖动
  • Lane 坏线位于边界位置
  • Refclk 切换时数据冲刷
  • LP/HS 快速切换
  • 电压摆幅极限

🔄 重传/流控

  • Replay Buffer 满+ACK 同时到达
  • 连续 NAK 风暴
  • Credit 0 时仍收到数据
  • ACK 与 NAK 乱序
  • 重传后原包与新包冲突

🌙 电源管理

  • PM 请求与数据并发
  • 退出 L2 时 Sideband 消息丢失
  • Retrain 与低功耗状态竞争
  • CLKREQ 抖动
  • 低功耗期间 CRC 错误

🧩 协议层

  • 非法/畸形 TLP
  • VC 与 TC 非法组合
  • Completion Timeout 风暴
  • AXI ID 冲突
  • 排序规则边界冲突

🧪 压力/异步

  • 复位与时钟切换并发
  • 多 Die 独立复位
  • 异步 CDC 边界亚稳态
  • 配置寄存器与链路状态竞争
  • 长时间满负荷 soak
